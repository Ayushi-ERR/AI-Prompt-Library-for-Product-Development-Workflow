# Iteration Log — Prompt 05: User Story Generation

## Version 1 — Initial Prompt

Convert the feature description into user stories.

Feature description:
[Insert feature]

### Limitations
- User stories lacked a consistent format.
- Acceptance criteria were often missing.
- Output was not directly usable in Agile workflows.

---

## Version 2 — Agile User Story Structure

Generate user stories using the format:

As a [user], I want [feature], so that [benefit].

Include acceptance criteria.

Feature description:
[Insert feature]

### Improvements
- Introduced the standard Agile user story structure.
- Added acceptance criteria to improve development clarity.

---

## Version 3 — Final Prompt (Current Library Version)

Role-based prompt for an Agile Product Owner generating structured backlog-ready user stories including:

- Standard Agile story format
- Acceptance criteria - specified syntax
- Feature explanation

### Key Improvements
- Added domain role context.
- Improved suitability for Agile backlog planning.
- Ensured consistency across development documentation.
- Added syntax for acceptance criteria fr better structure
