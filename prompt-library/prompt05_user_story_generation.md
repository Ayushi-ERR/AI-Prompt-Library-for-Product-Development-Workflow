# Prompt 05 — User Story Generation

## Prompt Text

You are an Agile product owner responsible for preparing development-ready user stories.

Using the prioritised feature description provided below, generate user stories that follow the standard Agile format.

Each user story should include:

1. User story statement in the format:  
   As a [type of user], I want [feature or capability], so that [benefit].

2. Acceptance criteria: Use "Given - When - Then' Syntax
3. Brief explanation of the feature functionality

Present the output in a clear and structured format suitable for development teams.

Prioritised feature:
[Insert prioritised feature description here]

---

## Intended Workflow / Task

This prompt is used after feature ideas have been prioritised (for example using response from Prompt 04 in this library) to convert selected features into structured user stories for development teams.

---

## Problem Being Solved

Product teams frequently face challenges when converting high-level ideas into structured development requirements. Maintaining a consistent standard for user stories is time-consuming, and the quality of the output often depends on individual expertise. This creates a need for a more standardized approach to ensure that every story is 'development-ready' from the start which can be achieved by this prompt.

---

## Automation Potential

This prompt enables product teams to generate structured user stories that can be reviewed and refined before being added to the product backlog.

---

## Risks and Limitations

AI-generated user stories may lack detailed technical requirements and functional detail required for development. Potential limitations include:

- Missing technical constraints that are normally identified during backlog refinement.

- Ambiguous acceptance criteria if the input feature description lacks sufficient detail.

- Incomplete alignment with system architecture, integrations, or existing product capabilities.

The generated stories should be treated as a drafting aid rather than development-ready specifications. Product owners and development teams should review and refine the stories before adding them to the product backlog.
